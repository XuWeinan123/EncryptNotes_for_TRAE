### Bug 修复记录

#### Bug 1：新建笔记后窗口不自动激活输入
- **根因**：`NSTextView` 创建后没有自动成为 first responder
- **修复**：创建 `AutoFocusTextView: NSTextView` 子类，重写 `viewDidMoveToWindow()`，在视图加入窗口后通过 `DispatchQueue.main.async` 调用 `window.makeFirstResponder(self)`

#### Bug 2：置顶按钮默认未启用，需按三次才生效
- **根因**：`@Published var isPinned: Bool = true` 在 ViewModel init 时立即触发 `$isPinned` sink，sink 调用 `togglePin()` 将 store 中的状态翻转成 false，导致 ViewModel 和 store 状态不一致
- **修复**：
  1. `MacNoteWindowStore` 新增 `setPinned(_:for:)` 方法直接设置状态
  2. ViewModel 初始化时从 store 读取真实 isPinned 状态（`@Published var isPinned: Bool` 无默认值）
  3. sink 中改用 `setPinned(newValue, for:)` 替代 `togglePin`
  4. `LockedStickyNoteView.togglePin()` 同步修复：先读取当前值再 set，避免计算属性读到翻转后的值

#### Bug 3：点击其他软件后置顶失效
- **根因**：
  1. `hidesOnDeactivate` 默认 true 导致窗口失活时隐藏
  2. `windowDidBecomeKey` 依赖错误的 store 状态（Bug 2 导致）
  3. 切换 Desktop/Space 时窗口不跟随
- **修复**：
  1. 设置 `window.hidesOnDeactivate = false`
  2. 监听 `NSApplication.didBecomeActiveNotification` 重新应用窗口层级
  3. 添加 `windowDidResignKey` 代理方法，置顶窗口失焦时保持 `.floating` 层级
  4. 置顶窗口设置 `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`，跨所有 Space/全屏保持可见

### UI 布局调整（第二次迭代）
- 默认窗口尺寸改为 400×250（居中屏幕），用户调整后记住尺寸（已通过 MacNoteWindowStore 持久化）
- 隐藏 macOS 默认红绿灯按钮（close/miniaturize/zoom）
- 左上角：自定义关闭按钮（xmark 图标）
- 右上角：锁图标（`lock.fill` 加密 / `lock.open.fill` 明文，disabled，hover 显示提示文字）→ 删除按钮 → 置顶按钮
- 移除底部「已保存」「同步中」等状态文字
- 仅在无网络时，右下角显示红色「无网络」文字
- 编辑区使用 NSScrollView + AutoFocusTextView 自动填充剩余空间，overlay 式滚动条
- 删除了不再使用的 `PlainStatusBadge.swift`

### 跨 Space 置顶实现要点
```swift
// StickyNoteWindowManager.applyWindowLevel
window.level = isPinned ? .floating : .normal
if isPinned {
    window.collectionBehavior.insert(.canJoinAllSpaces)   // 所有 Space 可见
    window.collectionBehavior.insert(.fullScreenAuxiliary) // 全屏 Space 可见
} else {
    window.collectionBehavior.remove(.canJoinAllSpaces)
    window.collectionBehavior.remove(.fullScreenAuxiliary)
}
```

## 遗留/待注意事项
- `StickyNoteWindow` 子类覆盖 `canBecomeKey`/`canBecomeMain` 返回 true，确保无边框/自定义窗口可正常成为 key window
- Debug 配置（`SealNoteDebug.entitlements` 为空）在 macOS 上同样不使用 iCloud，会自动落到本地存储
- 网络监测使用 `NWPathMonitor`，`isNetworkAvailable` 仅用于显示「无网络」提示，不影响本地保存
- 窗口位置/尺寸/置顶状态存 UserDefaults，key 为 `MacWindowStates`，value 为 [noteId: MacWindowState] JSON
