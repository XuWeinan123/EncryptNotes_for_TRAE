# Seal NotemacOS PRD v0.1

## 1. 产品范围

Seal NotemacOS v0.1 是一个常驻菜单栏的悬浮便利贴 App。

本版本只完成一个最小但完整的产品闭环：

> 用户可以通过菜单栏或全局快捷键快速创建一张默认置顶的便利贴；便利贴可以是明文笔记，也可以是加密笔记；笔记以独立 JSON 文件保存到 iCloud；加载正确密钥后，App 在本机解密并展示加密笔记；删除的笔记可以进入回收站并恢复。

v0.1 不追求完整笔记管理能力，不做 Markdown、Checklist、标签、搜索、颜色等扩展功能。

---

## 2. 产品定位

### 2.1 一句话定位

**一个支持 iCloud 同步与本地加密的 macOS 悬浮便利贴。**

### 2.2 核心价值

- 随时记录：不需要进入完整笔记 App。
- 默认悬浮：新建和打开的便利贴默认置顶。
- 明文与加密共存：普通内容快速记录，敏感内容本地加密。
- 文件归用户所有：通过 iCloud 文件同步，不依赖自建账号和服务器。
- 与Seal Note移动端共享同一套加密空间和数据文件。

## 5. 核心产品流程

### 5.1 新建明文笔记

```text
用户点击菜单栏或按下全局快捷键
→ 出现一张新的空白便利贴
→ 窗口默认置顶并进入编辑状态
→ 用户输入内容
→ App 自动保存为明文 JSON 文件
→ 文件写入 iCloud
```

### 5.2 新建加密笔记

```text
用户选择“新建加密笔记”
→ App 检查本机是否已加载密钥
→ 已加载：创建新的加密便利贴
→ 用户输入内容
→ App 在本机加密正文
→ 保存为加密 JSON 文件并写入 iCloud
```

如果本机未加载密钥：

```text
不创建笔记
→ 提示“加载密钥后才能创建加密笔记”
→ 提供“加载密钥文件”入口
```

### 5.3 打开最近笔记

```text
用户点击菜单栏图标
→ 菜单显示最近编辑的笔记
→ 用户点击某条笔记
→ 对应便利贴窗口出现并获得焦点
```

### 5.4 删除与恢复

```text
用户删除笔记
→ 文件移动到 iCloud trash 目录
→ 便利贴窗口关闭
→ 用户可在回收站恢复或永久删除
```

## 6. 菜单栏设计

菜单栏是 v0.1 的主要入口。

建议菜单结构：

```text
新建明文笔记                 ⌥⌘N
新建加密笔记                 ⌥⇧⌘N

最近笔记
  笔记第一行……
  笔记第一行……
  加密笔记 · 未加载密钥
  最多显示 8 条

全部笔记…
回收站…

加载密钥文件… / 移除本机密钥
设置…
退出Seal Note
```

### 6.1 最近笔记规则

- 按 `updated_at` 从新到旧排序。
- 最多显示 8 条。
- 明文笔记显示第一行非空文本。
- 加密笔记在密钥已加载时显示解密后的第一行。
- 加密笔记在未加载密钥时只显示：

```text
加密笔记 · 未加载密钥
```

- 未解锁的加密笔记不得生成明文预览缓存。

### 6.2 全部笔记

为避免超过 8 条后无法找回旧笔记，v0.1 提供一个基础笔记列表。

只支持：

- 查看全部笔记
- 按最近更新时间排序
- 打开笔记
- 删除笔记

不支持搜索、标签、分组和批量操作。

## 7. 快速新建与便利贴窗口

### 7.1 新建入口

v0.1 支持：

- 菜单栏新建
- 全局快捷键新建

默认快捷键：

| 操作 | 默认快捷键 |
|---|---|
| 新建明文笔记 | `⌥⌘N` |
| 新建加密笔记 | `⌥⇧⌘N` |

用户可以在设置中修改快捷键。

### 7.2 窗口规则

- 每条打开的笔记对应一个独立便利贴窗口。
- 新建后立即进入编辑状态。
- 窗口默认 Pin，保持在当前 Desktop 的普通应用窗口之上。
- 用户可以手动取消 Pin 或重新 Pin。
- 窗口默认出现在当前活跃 Desktop。
- v0.1 不承诺切换 Desktop 后继续显示；“所有 Desktop 显示”属于 P1。
- 关闭窗口只代表隐藏笔记，不代表删除。
- 再次打开时恢复该笔记上次的窗口尺寸和位置。

### 7.3 编辑器

v0.1 使用 macOS 原生文本编辑能力，优先保证中文输入法稳定。

必须支持：

- 简体中文输入法候选框正确跟随光标
- 拼音组合输入不丢字、不重复上屏
- 回车提交候选与换行行为正常
- 中英文混合输入
- 中文标点
- 复制、粘贴、撤销、重做

v0.1 只保存纯文本，不做 Markdown 实时渲染和富文本格式。

### 7.4 自动保存

- 用户输入后自动保存。
- 输入停止约 500 毫秒后触发保存。
- 窗口失去焦点或关闭时立即保存。
- 保存失败时显示非阻塞错误提示。
- 不提供手动“保存”按钮。

## 8. 笔记类型与文件设计

### 8.1 iCloud 目录

```text
iCloud Drive / Seal Note /
  vault.json
  notes /
    <note_id>.bkwplain.json
    <note_id>.bkwenc.json
  trash /
    <note_id>.bkwplain.json
    <note_id>.bkwenc.json
  meta /
```

每条笔记对应一个独立文件。

### 8.2 明文笔记

明文笔记适合普通临时内容。

规则：

- 内容以可读 JSON 保存。
- 文件可以被普通文本工具读取、复制和备份。
- App 必须明确显示“明文”状态，避免用户误以为内容已加密。

扩展名：

```text
.bkwplain.json
```

示例：

```json
{
  "version": 1,
  "app": "BieKanWo",
  "type": "plain_note",
  "note_id": "A7F3D9B2-82D1-4A41-9C22-31F9B21A901C",
  "created_at": "2026-06-26T10:00:00Z",
  "updated_at": "2026-06-26T10:05:00Z",
  "content": "需要记录的内容"
}
```

### 8.3 加密笔记

加密笔记适合敏感内容。

规则：

- 正文先在本机加密，再写入 iCloud。
- v0.1 沿用移动端的 AES-GCM 加密方案和加密文件结构。
- 文件扩展名为：

```text
.bkwenc.json
```

允许明文保存的最低限度字段：

- `version`
- `app`
- `type`
- `note_id`
- `vault_id`
- `created_at`
- `updated_at`
- `encryption.algorithm`
- `encryption.key_version`
- `encryption.nonce`

不得明文保存：

- 正文
- 标题
- 第一行预览
- 摘要
- 标签
- 搜索索引

## 9. iCloud 同步

### 9.1 同步规则

- 新建笔记：新增对应文件。
- 编辑笔记：更新对应文件及 `updated_at`。
- 删除笔记：将文件移动到 `trash` 目录。
- 恢复笔记：将文件移回 `notes` 目录。
- 永久删除：从 `trash` 目录删除文件。
- 离线时允许编辑；恢复联网后继续同步。

### 9.2 同步状态

v0.1 只提供三个状态：

- 已保存
- 正在同步
- 同步失败

不做复杂同步日志和版本历史。

### 9.3 同步冲突

多设备同时修改同一笔记时，不允许静默丢失任意一方内容。

v0.1 的处理方式：

- 保留更新时间较新的原笔记。
- 将另一份保存为新的“冲突副本”。
- 冲突副本生成新的 `note_id`。

## 10. 密钥加载

### 10.1 密钥文件

密钥文件扩展名：

```text
.bkwkey
```

密钥文件是打开加密笔记的唯一凭证。

### 10.2 加载流程

```text
用户选择“加载密钥文件”
→ 系统文件选择器打开
→ 用户选择 .bkwkey 文件
→ App 在本机读取并校验密钥
→ 校验成功后解密当前加密空间中的笔记
```

提示文案：

```text
密钥文件只会在本机读取，不会上传。
```

### 10.3 本机保存

- 加载成功后，密钥保存到本机 Keychain。
- 密钥不得写入 iCloud Drive。
- 密钥不得通过 iCloud Keychain 同步。
- 用户可以选择“移除本机密钥”。
- 移除后，加密笔记立即回到锁定状态。

### 10.4 错误密钥

导入错误密钥时：

- 不展示任何明文内容。
- 不修改现有笔记文件。
- 提示：

```text
无法使用这个密钥解锁当前加密空间。
```

### 10.5 首次创建加密空间

如果 iCloud 中不存在 `vault.json`：

- 用户首次新建加密笔记时，引导创建加密空间。
- App 在本机生成新密钥。
- 要求用户将 `.bkwkey` 文件保存到自己选择的位置。
- 用户确认已保存后，才允许创建第一条加密笔记。

v0.1 不提供密钥找回和客服恢复。

---

## 11. 未加载密钥状态

当 iCloud 中存在加密笔记，但当前 Mac 未加载密钥时：

- 全部笔记列表显示锁定占位项。
- 菜单栏最近笔记显示“加密笔记 · 未加载密钥”。
- 打开笔记后显示锁定窗口。
- 不允许查看或编辑正文。
- 提供“加载密钥文件”按钮。
- 允许将锁定笔记移入回收站，但需要二次确认。

锁定窗口文案：

```text
这台 Mac 还没有当前加密空间的密钥。
加载密钥文件后，笔记将在本机解密显示。
```

---

## 12. 回收站

### 12.1 删除规则

- 删除笔记时不立即永久删除。
- 文件移动到 iCloud `trash` 目录。
- 删除操作同步到其他设备。
- 删除后对应便利贴窗口关闭。

### 12.2 回收站功能

回收站支持：

- 查看已删除笔记
- 恢复
- 永久删除
- 清空回收站

v0.1 不做自动清理期限。

加密笔记在未加载密钥时只显示锁定占位，不显示正文预览。

## 14. v0.1 验收标准

### 14.1 菜单栏与快速新建

1. App 启动后在菜单栏显示入口。
2. 默认不显示 Dock 图标。
3. 用户可通过菜单栏和全局快捷键新建明文笔记。
4. 用户可通过菜单栏和全局快捷键新建加密笔记。
5. 新建窗口立即获得输入焦点。
6. 最近笔记按更新时间排序，最多显示 8 条。
7. 点击最近笔记可以打开并聚焦对应窗口。
8. 用户可以从“全部笔记”打开超过最近 8 条之外的旧笔记。

### 14.2 悬浮窗口与中文输入

1. 新建笔记默认 Pin。
2. Pin 状态下，窗口位于当前 Desktop 的普通应用窗口之上。
3. 用户可以取消 Pin 和重新 Pin。
4. 关闭窗口不会删除笔记。
5. 重新打开时恢复上次窗口尺寸和位置。
6. 系统简体中文输入法候选框正确跟随光标。
7. 拼音组合输入不丢字、不重复上屏。
8. 回车提交候选、换行和中英文混输行为正常。
9. 输入内容自动保存，关闭窗口前的最新内容不会丢失。
10. v0.1 切换 Desktop 后窗口不可见，不作为缺陷。

### 14.3 明文与加密

1. 明文笔记保存为独立 `.bkwplain.json` 文件。
2. 明文文件可被普通文本工具读取。
3. App 明确显示明文状态。
4. 加密笔记保存为独立 `.bkwenc.json` 文件。
5. 加密文件中不得出现正文、标题、预览或摘要明文。
6. 未加载密钥时不能查看或编辑加密正文。
7. 导入错误密钥时不能解密笔记。
8. 导入正确密钥后可以查看和编辑加密笔记。
9. 编辑后的内容重新加密后再写入 iCloud。
10. 移除本机密钥后，加密笔记立即恢复锁定状态。
11. 密钥文件和密钥内容不得上传到 iCloud Drive。

### 14.4 iCloud 与回收站

1. 新建笔记后，iCloud 中生成对应文件。
2. 修改笔记后，对应文件被更新。
3. 离线编辑后，恢复联网可以继续同步。
4. 同一 Apple ID 的其他设备可以看到新增或修改后的文件。
5. 并行编辑冲突不会静默丢失内容。
6. 删除笔记后，文件移动到 `trash` 目录。
7. 用户可以恢复、永久删除或清空回收站。
8. 未加载密钥时仍可删除或恢复加密文件，但不能查看正文。

## 15. 当前阶段最终目标

macOS v0.1 的目标不是做一个完整的桌面笔记工具，而是先验证以下闭环：

> 用户可以从菜单栏或全局快捷键快速创建一张默认置顶的便利贴；普通内容可以保存为明文笔记，敏感内容可以保存为加密笔记；所有笔记以独立 JSON 文件通过 iCloud 同步；正确加载密钥后，加密笔记只在本机解密；删除的笔记可以进入回收站并恢复。

这个闭环稳定后，再进入 P1，补齐“所有 Desktop 显示、隐私隐藏、透明度、折叠便签和开机启动”，完成真正的“跨桌面的加密悬浮便利贴”体验。
