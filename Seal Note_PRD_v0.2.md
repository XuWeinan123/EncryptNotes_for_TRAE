# Seal NotePRD v0.2

## 0. 文档说明

本文基于Seal NotePRD v0.1 与当前项目 `AGENTS.md` 中描述的 v0.1 代码现状整理，用于指导 v0.2 迭代。

v0.2 的核心变化是：产品不再是“必须导入密钥才能使用的加密笔记”，而是调整为“明文笔记 + 可选加密笔记”并存的卡片笔记 App。

用户可以直接创建明文笔记；也可以创建或导入密钥后创建加密笔记。未加载密钥时，加密笔记以乱码卡片存在；加载正确密钥后，乱码卡片恢复为可读内容。

本次迭代的 UI 与交互细化需要使用 `uiuxpromax` skill 进行设计优化，重点优化首页、侧边栏、密钥卡片、新建笔记开关、加密笔记锁定/解锁态、标签筛选、回收站与设置页。

---

## 1. v0.2 产品目标

### 1.1 一句话目标

让Seal Note成为一个可以自然使用的卡片笔记 App，同时保留“需要时加密”的能力。

### 1.2 核心目标

v0.2 要完成以下产品闭环：

```text
用户首次打开 App
→ 看到默认欢迎笔记与标签示例
→ 可以直接创建明文笔记
→ 首次创建笔记时提示创建密钥
→ 用户可跳过，继续创建明文笔记
→ 用户创建/导入密钥后，可创建加密笔记
→ 加密笔记在未加载密钥时显示为乱码卡片
→ 加载密钥后，加密笔记恢复为可读内容
→ 用户可通过侧边栏管理密钥、筛选标签、进入回收站
```

### 1.3 产品定位变化

v0.1：

```text
加密优先：没有密钥，不能正常使用笔记能力。
```

v0.2：

```text
记录优先：没有密钥，也能创建和管理明文笔记；密钥用于创建和查看加密笔记。
```

## 3. 核心产品模型

### 3.1 两类笔记

v0.2 中存在两类笔记：

| 类型 | 文件后缀 | 是否加密 | 是否需要密钥 | 适用内容 |
|---|---|---:|---:|---|
| 明文笔记 | `.bkwplain.json` | 否 | 否 | 普通想法、公开记录、低敏内容 |
| 加密笔记 | `.bkwenc.json` | 是 | 查看真实内容需要 | 私密内容、敏感想法、个人记录 |

### 3.2 明文笔记

明文笔记会真实以明文落盘。

这意味着：

- 明文笔记内容会保存在 `.bkwplain.json` 文件中。
- 该文件同步到 iCloud 后，文件内容可被直接读取。
- 明文笔记不适合保存敏感内容。
- 明文笔记不需要密钥即可查看、编辑、删除、恢复和筛选。

PRD 中必须明确告知用户：

> 明文笔记不会加密，适合普通内容；敏感内容建议使用加密笔记。

### 3.3 加密笔记

加密笔记会以 `.bkwenc.json` 保存。

当密钥未加载时：

- 卡片以乱码形式展示。
- 卡片显示上锁 icon。
- 可删除。
- 可进入回收站。
- 可从回收站恢复。
- 不可查看真实内容。
- 不可编辑真实内容。
- 不参与搜索。
- 不参与标签解析与标签统计。

当密钥已加载时：

- 卡片解密为可读内容。
- 卡片显示未上锁 / 已解锁 icon。
- 可查看、编辑、删除、恢复。
- 可参与搜索。
- 可参与标签解析与标签统计。

### 3.4 未解密加密笔记的产品原则

未解密加密笔记在产品表现上等价于“一张用乱码记录的明文卡片”。

核心理解：

```text
文件始终存在。
没有密钥时，用户看到的是乱码。
有密钥时，乱码恢复成真实内容。
```

示例：

用户 A 将一条未解密状态下看到的加密笔记文件发给用户 B。未来如果产品支持导入文件，用户 B 导入该文件后，只要使用用户 A 的对应密钥，就可以恢复出同样的笔记内容。

v0.2 不实现外部文件导入，但数据结构和交互表达不能阻断这个未来方向。

## 4. 默认笔记

### 4.1 创建时机

仅当满足以下条件时，App 自动创建默认笔记：

```text
首次启动
+ 当前 iCloud / 本地 vault 中没有任何 `.bkwplain.json` 或 `.bkwenc.json` 笔记文件
```

如果用户删除默认笔记，App 不应再次自动创建。

### 4.2 默认笔记类型

默认笔记全部为明文笔记。

原因：

- 新用户通常还没有创建密钥。
- 默认笔记用于说明基础用法，不应依赖密钥。
- 明文默认笔记可直接参与标签示例。

### 4.3 默认笔记内容建议

默认笔记 1：欢迎

```text
欢迎使用Seal Note。

你可以像写卡片一样记录想法，也可以在需要时创建加密笔记。

#欢迎 
```

默认笔记 2：标签说明

```text
标签写法示例：

在正文中输入 #灵感 或 #日记 ，它们会出现在侧边栏的标签区域。

#使用说明 #标签 
```

默认笔记 3：加密说明

```text
加密笔记适合保存更私密的内容。

首次创建笔记时，你可以选择创建密钥。创建密钥后，新建笔记时可以打开“加密笔记”开关。

#加密 
```

### 4.4 验收标准

- 新安装且无任何笔记时，自动出现默认笔记。
- 默认笔记为 `.bkwplain.json`。
- 默认笔记中的标签可被侧边栏识别。
- 用户删除默认笔记后，App 不应重复生成。
- 如果 iCloud 中已有任意笔记，不创建默认笔记。

## 5. 新建笔记

### 5.1 入口

首页继续使用浮动 `+` 按钮进入新建笔记 sheet。

新建页包含：

- 正文输入框
- “加密笔记”开关
- 保存按钮
- 取消按钮
- 必要的密钥状态提示

### 5.2 加密笔记开关

新建笔记页提供开关：

```text
加密笔记
```

状态含义：

| 开关状态 | 保存结果 |
|---|---|
| 关闭 | 保存为 `.bkwplain.json` 明文笔记 |
| 开启 | 保存为 `.bkwenc.json` 加密笔记 |

### 5.3 无密钥状态

当密钥未加载时：

- 新建页默认关闭“加密笔记”开关。
- 用户可正常创建明文笔记。
- 用户尝试打开“加密笔记”开关时，弹出密钥引导。
- 如果用户选择创建密钥，创建成功后自动加载密钥，并打开“加密笔记”开关。
- 如果用户选择导入密钥，导入成功后自动加载密钥，并打开“加密笔记”开关。
- 如果用户跳过或取消，开关保持关闭。

提示文案建议：

```text
创建密钥后，可以保存加密笔记。
密钥文件只会在本机读取，不会上传。
```

操作：

```text
创建密钥
导入密钥文件
继续写明文笔记
```

### 5.4 首次创建笔记的密钥提示

用户首次手动创建笔记时，App 应提示用户创建密钥。

触发条件：

```text
用户第一次点击新建笔记
+ 当前未加载密钥
+ 用户此前未处理过首次密钥提示
```

用户可以：

- 创建密钥
- 导入密钥文件
- 暂不创建，继续写明文笔记

如果用户选择“暂不创建”，本次新建笔记默认为明文笔记。

后续如果用户尝试开启“加密笔记”开关，需要再次提示创建或导入密钥。

### 5.5 创建密钥后自动加载

用户创建密钥成功后：

1. 生成新 `.bkwkey`。
2. 自动加载到当前设备。
3. 写入本机 Keychain。
4. 提示用户导出保存密钥文件。
5. 新建笔记页中的“加密笔记”开关自动打开。
6. 持久化默认新建模式调整为“加密”。

导出密钥不强制阻断流程。

### 5.6 默认新建模式的持久记忆

App 需要记忆用户上一次选择的笔记创建模式。

持久化字段建议：

```text
preferred_note_mode = plain | encrypted
```

规则：

| 场景 | 持久化状态变化 | 新建页默认状态 |
|---|---|---|
| 用户手动关闭加密开关 | 记为 `plain` | 明文 |
| 用户手动打开加密开关且密钥已加载 | 记为 `encrypted` | 加密 |
| 创建密钥成功 | 记为 `encrypted` | 加密 |
| 导入密钥成功 | 记为 `encrypted` | 加密 |
| 卸载密钥 | 记为 `plain` | 明文 |
| 重置密钥并生成新密钥 | 记为 `encrypted` | 加密 |
| App 启动时无密钥 | 若状态为 encrypted，应自动降级为 `plain` | 明文 |

重点原则：

> 持久记忆不能让用户进入不可用状态。只要当前没有密钥，新建笔记的实际默认模式必须是明文。

### 5.7 编辑笔记

编辑时不提供“明文 ↔ 加密”的转换能力。

v0.2 规则：

- 明文笔记编辑后仍为明文笔记。
- 加密笔记编辑后仍为加密笔记。
- 未解密加密笔记不可编辑。

原因：

- 避免 v0.2 引入转换、迁移、风险提示等复杂流程。
- 未来可单独设计“转为加密笔记 / 转为明文笔记”。

## 6. 首页与卡片状态

### 6.1 首页结构

v0.2 首页不再使用阻断式 locked / unlocked 首页。

首页始终显示笔记列表。

首页内容包括：

- 顶部栏
- 侧边栏入口
- 搜索入口，可选
- 当前标签筛选状态
- 笔记卡片流
- 浮动新建按钮

密钥状态只影响加密笔记的显示方式，不影响用户进入首页和创建明文笔记。

### 6.2 明文笔记卡片

明文笔记卡片展示：

- 正文摘要
- 标签高亮
- 创建/更新时间
- 可选操作菜单

明文笔记不显示锁 icon，或显示普通笔记标识。

### 6.3 已解密加密笔记卡片

已加载密钥时，加密笔记卡片展示：

- 未上锁 / 已解锁 icon
- 正文摘要
- 标签高亮
- 创建/更新时间
- 可选操作菜单

建议 icon：

```text
🔓
```

文案可选：

```text
加密笔记
```

### 6.4 未解密加密笔记卡片

未加载密钥时，加密笔记卡片展示：

- 上锁 icon
- 密文乱码
- 适度模糊效果
- 创建/更新时间
- 文件大小，可选
- “导入密钥后查看”提示，可选

建议表达：

```text
🔒 加密笔记
U2FsdGVkX1+f9aQ8zR1x...
7b3a9c11d8f04aa29f...
导入密钥后查看
```

设计要求：

- 乱码要可感知，但不要过度吓人。
- 可通过透明度、模糊、浅色背景降低压迫感。
- 不展示真实标签、摘要或搜索命中。
- 不对未解密加密笔记做特殊删除限制。

### 6.5 删除规则

首页中的所有笔记都可以删除：

- 明文笔记可以删除。
- 已解密加密笔记可以删除。
- 未解密加密笔记可以删除。

删除后进入回收站，不立即永久删除。

## 7. 密钥管理

### 7.1 密钥状态

v0.2 中密钥存在以下状态：

| 状态 | 说明 |
|---|---|
| 未创建/未加载 | 当前设备没有可用密钥 |
| 已加载 | 当前设备可以解密加密笔记 |
| 正在导入 | 用户正在选择并校验密钥文件 |
| 错误 | 密钥文件错误或无法解密当前加密笔记 |

### 7.2 创建密钥

入口：

- 首次创建笔记提示
- 新建页打开“加密笔记”开关时
- 侧边栏密钥卡片
- 设置页

创建密钥后：

- 自动加载
- 写入 Keychain
- 允许立即创建加密笔记
- 提示用户导出密钥文件
- 默认新建模式设置为加密

提示文案：

```text
密钥已经创建并加载。
请导出并妥善保存密钥文件。丢失密钥后，加密笔记将无法恢复。
```

### 7.3 导入密钥文件

文案统一使用：

```text
导入密钥文件
```

禁止使用：

```text
上传密钥文件
```

辅助说明统一为：

```text
密钥文件只会在本机读取，不会上传。
```

导入规则：

- 用户选择 `.bkwkey` 文件。
- App 在本机读取并校验。
- 如果当前存在加密笔记，应验证密钥能解密这些笔记。
- 导入成功后写入 Keychain。
- 导入成功后加密笔记恢复为明文内容。
- 导入成功后默认新建模式设置为加密。
- 导入失败不得进入已加载状态。

### 7.4 卸载本机密钥

“删除密钥”在产品文案上建议表达为：

```text
卸载本机密钥
```

含义：

- 只从当前设备移除密钥。
- 删除 Keychain 中的密钥。
- 不删除 iCloud 中的加密笔记文件。
- 不删除明文笔记。
- 加密笔记立即回到乱码态。
- 默认新建模式调整为明文。

卸载前提示：

```text
卸载后，这台设备将无法查看加密笔记内容。
你可以稍后重新导入密钥恢复查看。
建议先导出并保存密钥文件。
```

操作：

```text
导出密钥
继续卸载
取消
```

### 7.5 导出密钥

入口：

- 侧边栏密钥卡片
- 设置页
- 创建密钥后提示
- 卸载密钥前提示

导出文件名建议：

```text
Seal Note-密钥-2026-06-23.bkwkey
```

导出成功后提示：

```text
请妥善保存密钥文件。没有密钥，已加密笔记无法恢复。
```

### 7.6 重置密钥

重置密钥的含义：

```text
删除所有加密笔记
+ 删除旧密钥
+ 生成新密钥
+ 自动加载新密钥
```

删除范围：

- `notes/` 中所有 `.bkwenc.json`
- `trash/` 中所有 `.bkwenc.json`
- 当前设备 Keychain 中的旧密钥

保留范围：

- `notes/` 中所有 `.bkwplain.json`
- `trash/` 中所有 `.bkwplain.json`

重置后：

- 生成新 `.bkwkey`
- 自动加载新密钥
- 默认新建模式设置为加密
- 提示用户导出新密钥

风险提示：

```text
重置密钥将删除所有加密笔记，包括回收站中的加密笔记。
明文笔记会保留。
如果你还需要旧加密笔记，请先确认自己保存了旧密钥文件。
```

确认方式：

- 至少二次确认。
- 危险按钮使用红色语义 token。
- 不使用模糊文案，例如“清理”或“修复”。

## 8. 侧边栏

### 8.1 入口

首页左上角提供侧边栏入口。

侧边栏用于：

- 查看应用名称
- 进入设置
- 管理密钥
- 筛选标签
- 进入回收站

### 8.2 侧边栏结构

```text
侧边栏

1. 顶部区域
   - 应用名称
   - 设置按钮

2. 密钥卡片
   - 未加载密钥状态
   - 已加载密钥状态

3. 标签区域
   - 全部
   - #标签名称 + 数量

4. 底部区域
   - 回收站
```

### 8.3 顶部区域

展示：

```text
Seal Note
[设置]
```

## 9. 标签功能

### 9.1 标签解析规则

正文中以 `#` 开头，并以空格、换行或正文结尾结束的连续文本，记作标签。

示例：

```text
今天想到一个产品点 #产品 #隐私
```

识别为：

```text
#产品
#隐私
```

### 9.2 标签内容规则

- 标签必须以 `#` 开头。
- 标签内容不能为空。
- 标签内容不包含空格。
- 标签结束符包括：空格、换行、正文结尾。
- 不做次级标签。
- 不做标签层级。
- `#产品/隐私` 不拆分为父子标签，可作为普通标签 `产品/隐私` 处理。
- 标签区展示时统一带 `#` 前缀。

### 9.3 标签来源

只从可读笔记中解析标签。

可读笔记包括：

- 明文笔记
- 已加载密钥后的加密笔记

不可读笔记包括：

- 未加载密钥时的加密笔记

未解密加密笔记显示为乱码，不参与标签解析、标签统计和标签筛选。

### 9.4 标签筛选

点击侧边栏标签后：

- 首页进入标签筛选状态。
- 只展示包含该标签的可读笔记。
- 未解密加密笔记不展示。
- 回收站中的笔记不展示。

空状态文案：

```text
没有包含这个标签的可读笔记
```

如果因为卸载密钥导致当前标签下没有可读笔记，可以自动切回“全部”。

### 9.5 标签数量

标签数量展示该标签下当前可读、非回收站笔记数量。

动态变化场景：

- 新建明文笔记：更新标签数量。
- 新建加密笔记且密钥已加载：更新标签数量。
- 卸载密钥：移除加密笔记贡献的标签数量。
- 导入密钥：恢复加密笔记贡献的标签数量。
- 删除笔记：从标签数量中移除。
- 恢复笔记：重新加入标签数量。
- 永久删除：不再参与数量。
- 重置密钥：删除所有加密笔记贡献的标签数量。

## 10. 回收站

### 10.1 入口

回收站入口位于侧边栏最底部。

### 10.2 删除行为

删除笔记时，不立即永久删除，而是移动到回收站。

适用对象：

- 明文笔记
- 已解密加密笔记
- 未解密加密笔记

未解密加密笔记不做特殊处理。它和其他笔记一样可以被删除、恢复、永久删除。

### 10.3 保存时长

回收站中的笔记保留 30 天。

超过 30 天后自动永久删除。

自动清理触发时机建议：

- App 启动时
- 进入回收站页面时
- 从后台回到前台时，可选

### 10.4 回收站展示

回收站中的笔记按删除时间倒序展示。

明文笔记：

- 展示正文摘要
- 展示标签，可选
- 展示删除时间
- 展示剩余保留天数

已解密加密笔记：

- 展示未上锁 icon
- 展示正文摘要
- 展示标签，可选
- 展示删除时间
- 展示剩余保留天数

未解密加密笔记：

- 展示上锁 icon
- 展示乱码
- 可适度模糊
- 展示删除时间
- 展示剩余保留天数

### 10.5 恢复

回收站中的笔记允许恢复。

恢复规则：

- 明文笔记恢复后仍为明文笔记。
- 加密笔记恢复后仍为加密笔记。
- 未解密加密笔记恢复后仍为未解密加密笔记，加载密钥后可恢复内容。
- 当前 v0.2 没有文件夹结构，恢复后回到主列表。
- 数据结构需要预留未来文件夹恢复能力。

### 10.6 为未来文件夹预留

虽然 v0.2 不做文件夹，但回收站恢复需要预留未来字段。

建议删除时记录：

```json
{
  "deleted_at": "2026-06-23T12:00:00Z",
  "purge_after": "2026-07-23T12:00:00Z",
  "original_location": {
    "type": "root",
    "folder_id": null,
    "relative_path": null
  }
}
```

v0.2 中：

```text
original_location.type = root
```

未来增加文件夹后：

```text
original_location.type = folder
original_location.folder_id = <folder_id>
```

如果恢复时原文件夹不存在，回退到主列表。

### 10.7 永久删除

回收站支持：

- 单条永久删除
- 清空回收站

永久删除前需要二次确认。

提示文案：

```text
永久删除后无法恢复。
```

对于未解密加密笔记，不需要特殊文案。

### 10.8 重置密钥与回收站

重置密钥时，必须同时删除：

- 主列表中的所有加密笔记
- 回收站中的所有加密笔记

明文笔记保留：

- 主列表中的明文笔记保留
- 回收站中的明文笔记保留

## 18. 技术与实现注意事项

### 18.1 StoreKit / Paywall

v0.2 不做商业化。

实现上应：

- 移除 Paywall 触发路径。
- 移除新建笔记时的 Free 限制判断。
- `PurchaseStore` 不参与核心笔记流程。
- `Products.storekit` 可以暂时保留在工程中，但不应影响 UI 和业务逻辑。
- 测试用例中与 Free 限制相关的断言需要删除或改写。

### 18.2 VaultStore 状态机

v0.1 中的 noVault / locked / unlocking / unlocked / error 可以保留为内部技术状态，但不应直接决定首页是否可用。

产品层应抽象为：

```text
首页始终可用
密钥状态只影响加密笔记是否可解密
```

建议新增或调整衍生状态：

```text
isKeyLoaded: Bool
readableNotes: [Note]
lockedEncryptedNotes: [EncryptedNoteFile]
plainNotes: [PlainNote]
trashNotes: [TrashNote]
selectedTag: String?
```

### 18.3 解密失败

导入密钥时：

- 如果所有现有加密笔记都属于当前 vault，应要求全部可解密。
- 错 key 不能部分进入已加载状态。
- 解密失败不得泄露任何明文。
- 如果未来支持外部导入多来源加密文件，再设计多密钥或部分解密模型。v0.2 不做。

### 18.4 明文笔记安全提示

由于 v0.2 明文笔记真实明文落盘，产品需要在合适位置提示：

```text
明文笔记不会加密，适合普通内容。
敏感内容建议使用加密笔记。
```

不需要每次创建明文笔记都打扰用户，但首次引导和设置页必须说明。

### 18.5 回收站文件移动

删除时优先将文件移动到 `trash/` 目录。

如果 move 失败，再考虑 remove 或提示错误。

永久删除才真正删除文件。

### 18.6 原子写入

所有写入继续使用原子写：

```text
先写临时文件
再替换目标文件
```

避免 iCloud 同步中出现半写入文件。

### 18.7 多设备冲突

多设备冲突不得静默覆盖。

如果同一笔记在多设备被修改：

- 保留两个版本。
- 生成冲突副本。
- 不丢失用户内容。

## 20. v0.2 最终目标

v0.2 的最终目标是：

> 让Seal Note先成为一个可日常使用的卡片笔记 App，再把加密能力变成一个清晰、可选、可信的开关。

本版本完成后，用户应能自然理解：

- 我可以直接写普通笔记。
- 普通笔记是真明文，适合低敏内容。
- 我可以创建密钥来写加密笔记。
- 加密笔记没有密钥时就是乱码。
- 密钥加载后，乱码恢复成内容。
- 我可以随时卸载密钥，让加密笔记回到乱码态。
- 我可以用标签组织可读笔记。
- 删除的笔记会先进入回收站，30 天后才删除。
