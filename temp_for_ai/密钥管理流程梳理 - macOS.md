### 密钥管理流程梳理 - macOS

#### 基本原则

macOS 端的密钥决策集中在“设置 - 密钥”页。

Editor 和菜单栏遇到密钥问题时，只引导用户打开“密钥设置”，不在弹窗里直接加载、重新定位或移除密钥。

移除密钥引用表示用户放弃当前加密功能。移除完成后，应用内不应再存在加密笔记，包括回收站中的加密笔记。

应用默认前提是：应用内有且只有一种加密笔记。验证密钥时，如果存在加密笔记，只需要试解密其中一条即可；候选范围包括当前列表和回收站。

#### 状态定义

1. 未加载密钥，无加密笔记
2. 未加载密钥，有加密笔记
3. 已加载密钥，密钥可用，无加密笔记
4. 已加载密钥，密钥可用，有加密笔记
5. 已保存密钥引用，但密钥失效，无加密笔记
6. 已保存密钥引用，但密钥失效，有加密笔记

#### 1. 未加载密钥，无加密笔记

**设置页**

一行 List Item。左侧提示当前未加载密钥，右侧提供：创建新密钥、加载已有密钥。

**行为限制**

Editor 中尝试加密笔记时，提示需要先处理密钥，弹窗 1。

#### 2. 未加载密钥，有加密笔记

这种情况通常来自手动复制加密笔记或手动修改索引，是兜底场景。

**设置页**

List Item 样式同状态 1，但主操作优先引导用户加载已有密钥。

右侧提供：加载已有密钥、创建新密钥。

如果用户创建新密钥，提示弹窗 5。

如果用户加载的密钥无法解密现有加密笔记，提示弹窗 6。默认不保存这次选择的密钥。

**行为限制**

用户打开加密笔记并尝试解密时，提示弹窗 1。

#### 3. 已加载密钥，密钥可用，无加密笔记

**设置页**

一行 List Item。左侧显示密钥已加载，下方显示路径。右侧提供：打开密钥位置、移除密钥引用。

移除密钥引用时提示弹窗 2。

**行为限制**

无限制。

#### 4. 已加载密钥，密钥可用，有加密笔记

**设置页**

显示同状态 3。

移除密钥引用时提示弹窗 3。

**行为限制**

无限制。

#### 5. 已保存密钥引用，但密钥失效，无加密笔记

**设置页**

每次进入“密钥”页时刷新密钥状态。

密钥不可读取、路径已失效、内容被替换，或内容被修改，都视为密钥失效。

一行 List Item。左侧提示密钥失效，右侧提供：重新定位密钥、移除密钥引用。

其中“重新定位密钥”应作为主操作或更醒目的操作，方便用户从 Editor 弹窗跳转到设置页后继续处理。

移除密钥引用时提示弹窗 2。

**行为限制**

Editor 中尝试加密笔记时，提示需要先处理密钥，弹窗 4。

#### 6. 已保存密钥引用，但密钥失效，有加密笔记

**设置页**

显示同状态 5。

移除密钥引用时提示弹窗 7。

**行为限制**

用户打开加密笔记并尝试解密时，提示弹窗 4。

#### 密钥失效判断

保存密钥引用时，同时保存密钥路径、key_id，以及由 keyMaterial 解码后的原始字节计算出的 SHA-256 指纹。

之后如果路径下读不到密钥，或读到的密钥格式无效，视为密钥失效。

如果路径下密钥可读，但 key_id 或 SHA-256 指纹和保存时不同，视为密钥已被替换或内容已被修改，也归类为密钥失效。

如果存在加密笔记，密钥还需要能试解密其中一条加密笔记。试解密失败时，不保存新的密钥引用。

如果不存在加密笔记，只要密钥可读取且格式有效，就可以直接使用。

#### 弹窗

**弹窗 1：需要密钥**

用于未加载密钥时尝试加密或解密。

动作：打开密钥设置、取消。

**弹窗 2：移除密钥引用**

用于没有加密笔记时移除密钥引用。

动作：移除密钥引用、取消。

**弹窗 3：移除密钥引用前处理加密笔记**

用于密钥可用且存在加密笔记时。

动作：删除全部加密笔记（destructive）、先全部解密成明文、取消。

受影响范围必须说明包含当前列表和回收站中的加密笔记。

**弹窗 4：密钥失效**

用于保存了密钥引用但密钥不可用时，用户在 Editor 中尝试加密或解密。

动作：打开密钥设置、取消。

**弹窗 5：创建新密钥会影响现有加密笔记**

用于未加载密钥但应用内存在加密笔记时创建新密钥。

动作：删除这些加密笔记并创建新密钥（destructive）、取消。

受影响范围必须说明包含当前列表和回收站中的加密笔记。

**弹窗 6：所选密钥无法解锁现有加密笔记**

用于加载已有密钥，但该密钥无法解密现有加密笔记。

默认行为是不保存这次选择的密钥。

动作：重新选择密钥、删除这些加密笔记并使用此密钥（destructive）、取消。

受影响范围必须说明包含当前列表和回收站中的加密笔记。

**弹窗 7：密钥失效时移除密钥引用**

用于密钥失效且存在加密笔记时移除密钥引用。

动作：删除全部加密笔记（destructive）、取消。

受影响范围必须说明包含当前列表和回收站中的加密笔记。

#### 实现备注

macOS 的 `isKeyLoaded` 表示密钥可用，而不是仅表示存在已保存的密钥引用。

文件选择器和保存面板只接受当前密钥格式；旧密钥格式不再兼容。

新的加密正文只使用当前加密前缀；旧加密正文格式不再兼容。

移除密钥引用后，`preferredNoteMode` 回到明文。
